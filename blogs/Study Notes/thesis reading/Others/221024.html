    Title: High Precision Open-World Website Fingerprinting

    Author: Tao Wang from Hong Kong University of Science and Technology

    # Some Notions

    Notion Explanation
    WF Webiste Fingerprinting，浏览器指纹是一项识别用户浏览器的新技术，浏览器指纹可以通过浏览器对网站可见的配置、设置信息等来跟踪Web浏览器，以根据用户的各种特征来区分不同用户。
    Tor 第二代洋葱路由（onion routing）的一种实现，用户通过Tor可以在因特网上进行匿名交流。Tor的目标是通过隐藏路由信息和通信内容来提高用户的隐私。
    CW/OW closed-world：client只能访问一部分的网页(更简单) open-world：更加现实的情况，client访问的网页不受限
    TP/FP/TN/FN P和N表示预测的结果是positive还是negative，T和F表示真实结果和预测结果是否一致（True/False）
    recall 召回率/查全率，真实正样本中预测正确的概率，recall=TPR=TPTP+FNrecall=TPR=\frac{TP}{TP+FN}recall=TPR=​TP+FN​​TP​​
    precison 精确率/查准率，预测为正的样本中正确的概率，precision=TPTP+FPprecision=\frac{TP}{TP+FP}precision=​TP+FP​​TP​​
    FPR 真实负样本中预测错误的概率，FPR=FPFP+TNFPR=\frac{FP}{FP+TN}FPR=​FP+TN​​FP​​
    交叉验证 将拿到的训练数据，分为训练和验证集。将数据分成k份，其中一份作为验证集。然后经过k次(组)的测试，每次都更换不同的验证集。即得到k组模型的结果，取平均值作为最终结果。又称k-fold交叉验证

    # 摘要

    浏览网页时，对隐私敏感的用户会采用Tor等方式保护自己的隐私，但通过分析网络流量中的包序列、时间、顺序和方向等，用户隐私还是有可能发生泄露。

    传统的WF攻击（监视一组敏感网页，并识别客户端何时访问这些网页）在CW中对抗隐私保护技术Tor的效果显著。但在OW中，仅对high base rate pages能够成功。作者将base rate并入其引出的新的概念（r-precision）中，r较小时，传统的攻击效果较好，但当r很大（r>=1000）时，攻击效果很差。

    为了改善这一情况，作者提出了三种precision optimizer（Confidence-based PO、Distance-based PO、Ensemble PO）。将其应用于分类器，最终的预测效果可以得到明显提升。

    # 背景

    # 机器学习技术

    利用机器学习（有监督）得到一个分类器（classifier）

    • step1: 分类器接受一个输入（packet sequence(with timing, size and direction)）
    • step2：分类器判别输入是属于哪个分类（web page）
    • 分离器若识别到输入属于sensitive pages，且识别出具体来自哪一页面，则归为TP，否则，则归为WP，两者都不是则归为FP

    # 基本谬误（base rate fallacy）

    • 产生原因：高recall，低FPR模型拟合效果不一定好。client访问sensitive page的概率远小于FPR（open-world中sensitive pages占很少一部分），这时attacker会被错误的positive分类所淹没，无法区分client是否真的访问了sensitive page。
    • 解决方案：提高precision的定位，适度舍弃recall

    # 为什么优化precision比recall更重要

    • 基本谬误（base rate fallacy）
    • 寒蝉效应（chilling effect）：高precision下，即使是适度recall对client来说威胁更大。毕竟正样本的个数占少数。
    • 重复访问：用户往往频繁访问相同的网站。 低查全率、高精度攻击者最终能够确定client是否对特定敏感页面感兴趣

    # 实验设计

    # 数据集

    • 来源： Tor Browser（受欢迎且有弹性），Alexa页面，HTTP/1.1和HTTP/2均有
    • 组成： sensitive（monitored）和 non-sensitive（non-monitored）页面

    # precision和recall

    • precision: r=20和r=1000代表简单和困难分类，r越大，precision越小，预测越困难
    • recall: 0.2最小查全率，保证可以威胁到只访问一次sensitive page的client

    # 精度优化器（Precision Optimizers(POs)）

    POs将分类器视为黑盒，可以应用于所有的分类器。POs使分类器变得保守，以提高r-precision。工作原理如下：

    # Baseline precision

    无精度优化，只采用两种策略，用作其他优化器比较的基础。

    • Non-monitored class strategy
      思路: 增设一个non-monitored(non-sensitive) class，试图将non-sensitive pages更细地划分
      结果: 在更难的π1000\pi_{1000}π​1000​​场景中，所有攻击都是不精确的（采用了交叉验证）。

    • k-neighbors strategy
      思路：当对测试元素进行分类时，分类器找到与测试元素最接近的k个训练元素。只有当所有k个最接近的训练元素都属于一个正类时，分类器才会输出一个正类。否则，分类器将输出否定类。
      结果：难以实现高精度。

    # Confidence-based PO

    • 思路：定义一个匹配函数 match(P,C)match(P,C)match(P,C)，参数：输入P、所有训练类别C。当匹配函数值最大时，输出对应的类别作为分类结果；同时规定满足一定条件时（分类结果可信度不高），可以拒绝分类。关键在于匹配函数的定义方式。
    • 结果：精度和无优化相比有明显提高，但π1000\pi_{1000}π​1000​​情况下，精度仍不足。

    # Distance-based PO

    • distance: 在机器学习中，我们经常需要计算样本之间的差异，进而评价个体的相似性和类别等信息。特征空间中两个样本之间的距离就是两个样本相似性的一种反映。
    • Too-far PO和Too-close PO 两种思路。
      Too-far PO：测试包序列到假定类的距离大于期望类内距离的MfarM_{far}M​far​​倍，我们拒绝分类。
      Too-close PO: 如果至少有m个McloseM_{close}M​close​​类比它假定的类更接近包序列，我们拒绝分类。
    • 结果：π20\pi_{20}π​20​​和π1000\pi_{1000}π​1000​​情况下，效果都很好。

    # Ensemble PO

    • 思路：Ensemble Learning 在同一个训练集上分别训练了所有的分类器。 然后，取分类器的一个子集，对于每个测试元素，要求每个分类器确定假定的类。 当选择的分类器子集中的所有分类器之间没有一致的决定时，则拒绝分类。 因此，分类器越多，分类就越保守。

    # 识别敏感客户

    • 任务：通过分析client浏览的网页情况，来确认client是否为一类敏感群体（如政治派别）。

    # 攻击防御手段

    WTF-PAD开销最小但防御效果最差，Tammaraw防御效果最好但开销大。基于Confidence-based PO优化之后，面对Tammaraw精度提高5倍但仍然很低，但面对其他两个可以做到几乎不损失精度。

    # 攻击不同数据集

    • HTTP/1.1和HTTP/2
      应用优化器后两个数据集的precison明显提高，recall下降（这是必然）。两个数据集的precision几乎没有差异，而HTTP/1.1数据集的precision略高。

    • 同域页面（Same-domain Pages）
      任务：测试WF是否能够精确地分离相同类型和域的Web页面。
      结果：应用优化器不能在这个具有挑战性的问题上达到高精度，但确实带来了三到四倍的精度提高。

    • 不同的sensitive Pages
      简而言之，优化器可以应用于对各种网页的攻击（不限于Alexa）

    • Active Client
      任务：测试面对浏览网页会产生随机事件（如点击链接Link、滚动页面Ajax）的客户时的精度

      将这四类作为sensitive page替换原数据集sensitive pages中的四个，发现这四个新的类很难分类，虽然总体上不影响精度。
