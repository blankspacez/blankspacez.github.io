    # Docker基本原理

    • 镜像（image）： Docker 镜像就像软件安装包（比如 .exe 安装文件或 .iso 系统镜像）。它是一个模板，包含了应用程序及其运行环境（如操作系统、依赖库等）。
    • 容器（container）：容器是镜像的实例，它是一个可运行的实例，包含了应用程序和运行环境。容器可以启动、停止、删除、迁移等。
    • 仓库（repository）：仓库是镜像的存储库，它包含多个镜像。仓库可以是公共的或私有的（如企业内部的仓库）。 Docker官方提供的Docker Hub (opens new window) 是一个公共的仓库，它包含许多官方的镜像。

    # Docker常用命令

    • 拉取镜像 docker pull (opens new window)

    在Docker Hub上搜索你想要安装的镜像，然后使用docker pull命令拉取镜像，下面以安装nginx镜像为例，不指定版本号，默认使用latest版本。

    docker pull nginx:latest
    1
    • 查看镜像 docker images (opens new window)

    查看镜像是否拉取成功，使用docker images命令查看镜像列表。

    docker images
    1
    • 保存镜像 docker save (opens new window)

    使用docker save命令，将镜像保存到本地tar文件中, 这样做的好处有：1. 将镜像备份到本地，防止镜像丢失，方便后续恢复；2. 在离线状态可以将镜像传输到其他机器上；3. 方便将镜像推送到远程仓库。

    docker save -o nginx.tar nginx:latest

ll # 查看文件是否保存成功
    1
    2
    3
    • 删除镜像 docker rmi (opens new window)

    保存到本地之后可以使用docker rmi命令删除镜像

    docker rmi nginx:latest
    1
    • 加载镜像 docker load (opens new window)

    想要将删除的镜像重新加载到本地，可以使用docker load命令。

    docker load -i nginx.tar
    1
    • 运行镜像 docker run (opens new window)

    使用docker run命令运行镜像，以安装nginx镜像为例。

    docker run -d --name nginx -p 80:80 nginx:latest
    1
    • 查看容器 docker ps (opens new window)

    使用docker ps命令查看容器状态列表。

    docker ps # 显示正在运行的容器
docker ps -a # 显示所有容器
    1
    2
    • 进入容器 docker exec (opens new window)

    可以使用docker exec命令进入容器内部，方便进行编辑配置文件、修改环境变量等操作。

    docker exec -it nginx bash # 进入容器并打开bash终端
    1
    • 停止容器 docker stop (opens new window)

    使用docker stop命令停止容器。

    docker stop nginx
    1
    • 开启容器 docker start (opens new window)

    使用docker start命令开启容器。

    docker start nginx
    1
    • 删除容器 docker rm (opens new window)

    使用docker rm命令删除容器, 删除容器之前需要先停止容器。

    docker rm nginx

    # 数据卷

    数据卷（volume）: Docker 容器和宿主机之间的一个目录映射（虚拟目录、逻辑概念）。

    数据卷的作用：

    • 一方面，容器的文件系统是临时的，当容器删除或重建时，容器内部的数据会丢失。数据卷提供了一种方式，使数据能够独立于容器的生命周期存在，即使容器删除或重建，数据仍然保留。

    • 另一方面，容器中只包含了程序运行的最基本环境，数据卷使容器能够使用宿主机的数据。

    下图中，将Nignx容器下的目录映射到宿主机的目录，实现了这两个目录中数据的双向绑定。

    👉 常用命令 👇

    docker volume --help # 查看volume下的所有命令
    1
    docker inspect # 查看容器下的所有数据卷信息

    • 实现数据卷和容器目录间的映射，挂载数据卷（挂载完成后宿主机目录会默认创建）
    docker rm -f nginx # 需要先删除容器

docker run -d --name nginx -p 80:80 -v html:/usr/share/nginx/html nginx:latest # 挂载数据卷

    docker volume ls

    docker volume inspect html

    • 实现宿主机目录和容器目录的直接映射（事先创建好自定义宿主机目录）
    docker run -d \
    --name mysql \
    -p 3306:3306 \
    -e TZ=Asia/Shanghai \
    -e MYSQL_ROOT_PASSWORD=123456 \
    -v /root/mysql/data:/var/lib/mysql \
    -v /root/mysql/conf:/etc/mysql/conf.d \
    -v /root/mysql/init:/docker-entrypoint-initdb.d \
    mysql:latest

    # 构建镜像

    以构建Java应用程序的镜像为例：

    镜像分为很多层，已经构建好的层可以重复使用（无需二次下载），从而提高构建速度。

    编写Dockerfile文件：

    # 选择基础镜像（使用 OpenJDK 17，可根据需求修改）
FROM openjdk:17-jdk-slim

# 复制本地构建好的 Jar 包到容器内的/app.jar中
COPY xxx-demo.jar /app.jar

# 暴露端口（如 Spring Boot 默认端口 8080）
EXPOSE 8080

# 运行 Java 应用程序
ENTRYPOINT ["java", "-jar", "/app.jar"]

    # 自定义网络

    Docker 默认使用一个名为 bridge 的网络，该网络将容器连接到一个虚拟网卡上，以实现容器之间的通信。但是，默认网络中的容器之间进行通信时，需要使用容器的 IP 地址进行通信，这会增加网络的复杂性和维护成本。因此，我们可以自定义网络，同一个自定义网络中的容器之间可以通过容器名进行通信，从而简化网络配置和操作。

    docker network create my-network # 创建自定义网络
docker network connect my-network nginx # 将nginx容器连接到自定义网络
docker run -d --name mysql --network my-network mysql:latest # 在创建mysql容器时连接到自定义网络
docker network inspect my-network # 查看自定义网络详细信息
    1
    2
    3
    4

    运行以上命令后，nginx容器和mysql容器连接到同一个自定义网络my-network，这两个容器现在可以通过容器名进行互相通信，大大简化操作复杂性。
